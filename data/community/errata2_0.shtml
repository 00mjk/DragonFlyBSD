Errata Page for DragonFly 2.0.x

General note on errata: We do not roll a release CD every time we bump the sub-version. The standard upgrade method is to build from sources. The build manual page gives you information on how to upgrade your system from sources. See the download page for information on tracking changes with cvsup.

2.0.0 Errata

    There is a known bug with the installer. The installer can seg-fault when attempting to setup a European time zone. The work around is to setup a random time zone and fix /etc/localtime later, after you reboot into the hard drive.

    When newfs_hammer is run on a small partition it creates an UNDO area which is too small, risking an assertion during normal filesystem operation. The suggested minimum is 100M and can be set with the -u option to newfs_hammer.

    /usr/Makefile specifies the wrong tags for the 'release-src-cvsup' target. The work-around is to edit it to use the correct tag, 'DragonFly_RELEASE_2_0_Slip'.
